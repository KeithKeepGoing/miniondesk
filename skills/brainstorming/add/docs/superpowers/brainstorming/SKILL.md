## Skill: Brainstorming

When a task is vague or open-ended, use this skill BEFORE implementing:

1. Ask 3-5 clarifying questions to understand the requirement
2. Propose 2-3 different approaches with trade-offs
3. Save the chosen design to `/workspace/group/docs/designs/{task-name}.md`
4. Only proceed to implementation after design is confirmed

Example trigger phrases: "help me think", "brainstorm", "design a solution", "what are my options"
